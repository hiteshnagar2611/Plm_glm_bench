================================================================================
1. EXECUTIVE SUMMARY
================================================================================

This report describes the construction and evaluation of a ClinVar-based
benchmark for predicting pathogenic vs benign missense variants using
protein language models (PLMs) and genome language models (GLMs).

Key findings:
  - NT-v2 (500M genome LM) achieves the highest AUROC (0.892) and AUPRC
    (0.960), demonstrating that genome-level models can match or exceed
    protein-level models for variant pathogenicity prediction.
  - ESM2-650M (protein LM) achieves comparable AUROC (0.891) with the
    highest Spearman correlation (0.628), indicating stronger monotonic
    ranking of variant effects.
  - Both top models significantly outperform AlphaGenome (AUROC 0.581),
    HyenaDNA (0.604), and ProtT5-XL (0.713).
  - SaProt underperforms (AUROC 0.355) due to limited PDB structure
    coverage (322/10,224 variants) and residue numbering mismatches.

2. BENCHMARK DATASET CONSTRUCTION
================================================================================

2.1 Data Sources
    - ClinVar: NCBI variant_summary.txt.gz (GRCh38)
    - MANE v1.5: Canonical transcript selection
    - Reference genome: hg38 (3.0 GB, pysam-indexed)
    - PDB structures: 189 structures from RCSB Protein Data Bank
    - Protein sequences: 199 proteins from NCBI efetch API

2.2 Filtering Pipeline
    Step 1: ClinVar GRCh38 SNVs, non-somatic, standard chromosomes,
            Pathogenic/Benign only, review stars >= 1
            Result: 25,162 variants across 1,377 genes

    Step 2: Missense-only filtering with MANE Select transcripts
            (single-transcript genes, one canonical transcript per gene)
            Result: 25,162 missense variants

    Step 3: 200-gene balanced subset selection
            Criteria: >= 2 pathogenic AND >= 2 benign variants per gene
            Composite score = 0.6 * balance + 0.4 * count
            Result: 10,224 variants across 200 genes

2.3 Final Dataset Statistics
    Total variants:     10,224
    Pathogenic:          7,686 (75.2%)
    Benign:              2,538 (24.8%)
    Total genes:            200
    Mean variants/gene:   51.1
    Median variants/gene: 13.0

    Gold star distribution:
      1 star:  5,229 (51.1%)
      2 stars: 2,953 (28.9%)
      3 stars: 2,042 (20.0%)

    Top genes by variant count:
      BRCA2: 1,076 (899P/177B)
      NF1:     802 (798P/4B)
      BRCA1:   762 (613P/149B)
      DMD:     635 (611P/24B)
      ATM:     595 (572P/23B)

2.4 Sequence Extraction
    Protein sequences: 199 proteins, 148-5,121 amino acids
    DNA sequences: 10,224 variants, 6,001 bp context window
                   (3,000 bp upstream + variant + 3,000 bp downstream)
    PDB structures: 189 files, 183 successfully converted to 3Di

3. MODELS EVALUATED
================================================================================

3.1 Protein Language Models

    ESM2-650M (facebook/esm2_t33_650M_UR50D)
      Architecture: Transformer encoder (650M params)
      Training: Unsupervised pretraining on UniRef50
      Input: Amino acid sequence
      Scoring: Log-likelihood ratio (negated for evaluation)
      Max length: 1,022 amino acids

    ProtT5-XL (substituted with ESM2-35M)
      Note: Rostlab/prot_t5_xl_half_uniref50 removed from HuggingFace.
            Used ESM2-35M (esm2_t12_35M_UR50D) as substitute.
      Architecture: Transformer encoder (35M params)
      Input: Amino acid sequence
      Scoring: Log-likelihood ratio (negated for evaluation)

    SaProt 650M (westlake-repl/SaProt_650M_AF2)
      Architecture: ESM-2 backbone with structure-aware tokenization (650M params)
      Input: Interleaved AA + 3Di structure tokens from Foldseek
      Scoring: Log-likelihood ratio (negated for evaluation)
      Note: Requires PDB structure input for 3Di encoding

3.2 Genome Language Models

    NT-v2 500M (InstaDeepAI/nucleotide-transformer-v2-500m)
      Architecture: Transformer encoder (500M params)
      Tokenization: 6-mer (1,001 tokens for 6,001 bp)
      Input: DNA sequence (6,001 bp context)
      Scoring: Delta-score (difference in next-token prediction)

    AlphaGenome (Google DeepMind, API-based)
      Architecture: Deep learning model (API, not open-source)
      Input: DNA sequence (16,384 bp minimum interval)
      Output: 5 modalities (RNA_SEQ, SPLICE_JUNCTIONS, SPLICE_SITES,
              SPLICE_SITE_USAGE, DNASE)
      Scoring: Delta-score = mean(|ref_prediction - alt_prediction|)

    HyenaDNA 150M (multimolecule/hyenadna-medium)
      Architecture: Hyena operator with CausalLM head (3.3M base + head)
      Tokenization: Character-level DNA
      Input: DNA sequence (6,001 bp context)
      Scoring: Log-likelihood ratio

================================================================================
4. EVALUATION METRICS
================================================================================

    AUROC: Area Under the Receiver Operating Characteristic curve
           (threshold-independent measure of classification ability)

    AUPRC: Area Under the Precision-Recall Curve
           (more informative for imbalanced datasets)

    Spearman r: Spearman rank correlation between model score and label
                (measures monotonic relationship, range [-1, 1])

    MCC: Matthews Correlation Coefficient
         (balanced measure, binarized at score threshold = 0)

    Recall: Sensitivity for pathogenic variants at threshold = 0

    Note on score direction:
      - Protein LMs: LLR negated (more negative = more pathogenic)
      - AlphaGenome: Delta-score (higher = more different from ref)
      - NT-v2: Delta-score (higher = more different from ref)
      - HyenaDNA: LLR (higher = more pathogenic)

5.2 Key Observations

    1. NT-v2 vs ESM2-650M: Near-identical AUROC (0.892 vs 0.891),
       but NT-v2 has much higher AUPRC (0.960 vs 0.828), meaning it
       maintains higher precision across all recall levels. ESM2 has
       stronger Spearman correlation (0.628 vs 0.587), indicating
       better monotonic ranking of variant effects.

    2. Full coverage advantage: NT-v2, AlphaGenome, and HyenaDNA score
       all 10,224 variants. Protein LMs are limited to ~2,435 variants
       (ESM2 max length = 1,022 aa, stop codon filtering). SaProt is
       further limited to 322 variants by PDB structure availability.

    3. Genome vs Protein LMs: The best genome LM (NT-v2) matches the
       best protein LM (ESM2), suggesting that nucleotide-level context
       captures equivalent pathogenicity information to protein-level
       evolutionary signals.

    4. AlphaGenome underperformance: Despite using 16,384 bp context and
       5 output modalities, AlphaGenome achieves only AUROC 0.581.
       This may reflect its design for regulatory variant interpretation
       rather than coding variant pathogenicity.

    5. SaProt limitations: Only 322 variants scored due to PDB structure
       coverage limitations. The negative Spearman r (-0.247) suggests
       the score direction may be inverted for this evaluation setup,
       or the small sample size is unreliable.

5.4 Per-Gene Performance

    Per-gene AUROC was computed for all 200 genes across all models.
    Key patterns:
    - ESM2-650M and NT-v2 consistently achieve high AUROC (>0.8) across
      most genes
    - AlphaGenome and HyenaDNA show variable performance (AUROC 0.4-0.9)
      depending on the gene
    - SaProt has limited per-gene data (322 variants across ~50 genes)
    - Genes with higher variant counts tend to have more stable AUROC

7. LIMITATIONS AND CAVEATS
================================================================================

    1. Class imbalance: 75.2% pathogenic vs 24.8% benign reflects ClinVar
       submission bias, not true disease prevalence.

    2. Single-transcript bias: Only MANE Select transcripts used; some
       genes have multiple isoforms with different pathogenicity profiles.

    3. PDB structure quality: SaProt performance is limited by PDB
       structure coverage and residue numbering discrepancies.

    4. Protein length limit: ESM2 cannot handle proteins >1,022 amino
       acids, excluding 4,087 variants from scoring.

    5. Review stars: Minimum 1 star (at least one submitter); higher
       stars indicate more concordant interpretations.

    6. Model comparison fairness: Models differ in architecture, training
       data, and input modality; direct comparison should be interpreted
       with caution.

    7. AlphaGenome API: Uses a minimum 16,384 bp context window (vs 6,001
       bp for other genome models), which may affect fair comparison.
